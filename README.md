# Бесплатный мониторинг госзакупок на zakupki.gov.ru

Расширение Chrome для мониторинга закупок на закупки.гов.ру через RSS. Уведомления о новых тендерах приходят прямо в браузер. Без регистрации, без подписок, без передачи ваших фильтров куда-либо. Open source, MIT.

> Альтернатива Feeder, Tenderhelp, RTS-Tender и платным сервисам мониторинга госзакупок. Если задача — видеть новые лоты по нескольким сохранённым поискам, расширения хватит.

## Что умеет расширение

- Уведомления о новых закупках. Расширение опрашивает RSS-ленты zakupki.gov.ru с настраиваемым интервалом (от 1 минуты до недели, по умолчанию 10 минут) и шлёт системное уведомление, когда появляется новая запись. Клик по уведомлению открывает страницу закупки.
- Любые разделы zakupki.gov.ru. Работает с расширенным поиском (`/epz/order/extendedsearch/rss`), реестром контрактов, реестром жалоб ФАС, РНП и планами-графиками. Везде, где сайт даёт RSS-кнопку. Парсер совместим с RSS 2.0 и Atom.
- Без дублей. Каждая запись запоминается по `<guid>` или `<id>`, поэтому повторного уведомления не будет, даже если расширение перезапустится или вы поменяете интервал.
- Без слежки. Никаких аналитик, никаких сторонних серверов. Все данные хранятся локально в `chrome.storage.local`. Код открыт.
- Без регистрации. Установил, вставил RSS-ссылку, пользуешься. Никаких аккаунтов и платных тарифов.

## Установка

Расширение не содержит предустановленных RSS-лент. Какие поиски и реестры zakupki.gov.ru отслеживать, вы решаете сами. Никакого шаринга фильтров между пользователями нет.

### Способ 1. Скачать готовый ZIP из GitHub Releases (рекомендуется)

1. Откройте [страницу релизов](https://github.com/stufently/free-zakupki-gov-ru-monitor/releases) и скачайте последний `free-zakupki-gov-ru-monitor-X.Y.Z.zip`.
2. Распакуйте архив в любую папку, например `~/zakupki-monitor`.
3. Откройте `chrome://extensions` (или `edge://extensions`, `brave://extensions`).
4. В правом верхнем углу включите тумблер «Режим разработчика» (Developer mode).
5. Нажмите «Загрузить распакованное расширение» (Load unpacked) и выберите распакованную папку.
6. В списке появится «Бесплатный мониторинг госзакупок». Закрепите иконку расширения через меню Extensions.
7. Кликните на иконку, откройте «Настройки», добавьте RSS-ссылку с zakupki.gov.ru и сохраните.

### Способ 2. Через git clone (для разработчиков и контрибьюторов)

```bash
git clone https://github.com/stufently/free-zakupki-gov-ru-monitor.git
cd free-zakupki-gov-ru-monitor
```

Дальше шаги 3–7 как выше, но в качестве папки расширения выберите `extension/` внутри клонированного репозитория. Удобно тем, что `git pull` подтянет обновления, останется только перезагрузить расширение в `chrome://extensions` (кнопка обновления на карточке).

### Способ 3. Chrome Web Store

В работе. Опубликую после стабилизации интерфейса и первых отзывов. [Подписаться на релизы](https://github.com/stufently/free-zakupki-gov-ru-monitor/releases), чтобы получить уведомление о публикации.

### Обновление до новой версии

- Скачали ZIP — скачайте свежий, распакуйте поверх или в новую папку, на карточке расширения нажмите кнопку обновления.
- Через git — `git pull` в папке проекта и кнопка обновления на карточке расширения.

## Как получить RSS-ссылку с zakupki.gov.ru

Сайт zakupki.gov.ru даёт RSS-фид на любой результат расширенного поиска. Это и есть бесплатный официальный способ мониторинга госзакупок без API-ключей.

1. Откройте [zakupki.gov.ru](https://zakupki.gov.ru/) и нажмите «Расширенный поиск».
2. Настройте фильтры под свою задачу: ключевые слова (например, «охрана» или «ИТ-услуги»), регион, способ закупки, диапазон НМЦК, заказчик, ОКПД2.
3. Нажмите «Применить» и прокрутите вниз результатов поиска.
4. Под результатами найдёте кнопку «RSS» (оранжевая иконка). Кликните правой кнопкой и скопируйте адрес ссылки.
5. Вставьте эту ссылку в настройках расширения.

URL выглядит примерно так:

```
https://zakupki.gov.ru/epz/order/extendedsearch/rss?searchString=...&morphology=on&pageNumber=1&...
```

Можно подключить несколько лент. Например, отдельно по регионам, отдельно по ОКПД, отдельно по конкретным заказчикам. Расширение опрашивает каждую независимо и шлёт уведомления отдельно.

### Какие ещё RSS есть на zakupki.gov.ru

| Что отслеживается | Где находится RSS |
|---|---|
| Новые закупки (44-ФЗ, 223-ФЗ) | Расширенный поиск, кнопка RSS внизу страницы |
| Реестр контрактов | Реестр контрактов, расширенный поиск, кнопка RSS |
| Реестр жалоб (ФАС) | Реестр жалоб, расширенный поиск, кнопка RSS |
| Реестр недобросовестных поставщиков (РНП) | РНП, расширенный поиск, кнопка RSS |
| Планы-графики | Планы-графики, расширенный поиск, кнопка RSS |

Сайт zakupki.gov.ru периодически меняет интерфейс. Если кнопка «RSS» переместилась, ищите её в блоке экспорта результатов или в меню «Сохранить поиск».

## Зачем мониторить госзакупки

На zakupki.gov.ru ежедневно публикуются тысячи новых лотов. Если у вас:

- Поставка товаров или услуг как основной канал продаж. Платные сервисы стоят 5–50 тысяч ₽/мес. Если устойчивых поисков немного, RSS и это расширение закрывают задачу бесплатно.
- Маркетплейс или дистрибуция, и нужно отслеживать спрос по конкретным ОКПД2.
- Юридическая практика по госзаказу. Тогда полезен мониторинг жалоб ФАС и РНП по списку клиентов.
- Журналистика или аналитика — отслеживание закупок отдельных ведомств или регионов.
- Прототип Telegram-бота или внутренней рассылки про тендеры, и не хочется поднимать сервер.

Расширение даёт оповещения о новых государственных закупках без сервера, бота и подписки. Установили в браузер, добавили RSS-ссылки — дальше работает само.

## Сравнение с альтернативами

| | Это расширение | Платные сервисы (от 5 000 ₽/мес) | Telegram-боты | Feeder / RSS-ридеры |
|---|---|---|---|---|
| Цена | Бесплатно | 5 000 – 50 000 ₽/мес | Часто условно бесплатно с лимитами | Бесплатно |
| Регистрация | Не нужна | Аккаунт + KYC | Аккаунт Telegram | Часто не нужна |
| Скорость уведомлений | от 1 минуты, настраивается | Минуты | Минуты | 15 мин – 1 ч |
| Долгие URL zakupki.gov.ru | Работают | Работают | Работают | Часто обрезаются |
| Хранение фильтров | Локально | На сервере провайдера | На сервере бота | Локально |
| Подходит для бизнес-критичных лотов | Для информирования — да. Для торгов в реал-тайме лучше платный сервис | Да | Зависит | Не очень |
| Open source | Да | Нет | Иногда | Зависит |

Платный сервис нужен, если важно реагировать в первые минуты и нужны фильтры по «сложным» полям, которые в RSS не передаются: тип обеспечения, динамика средней цены, ОКПД2 с группировкой и так далее. Для остальных задач RSS и это расширение покрывают сценарий.

## FAQ

### Это безопасно? Куда уходят мои поисковые запросы?

Никуда. Расширение не отправляет данные на сторонние серверы. RSS-ссылки и история найденных закупок хранятся в `chrome.storage.local` на вашем компьютере. HTTP-запросы идут только напрямую на zakupki.gov.ru. Код открыт, проверить можно в папке `extension/`.

### Будет ли работать в Edge, Yandex Browser, Brave, Opera?

Да. Все эти браузеры на Chromium и поддерживают расширения Chrome MV3. Установка через тот же режим разработчика.

### А Firefox?

Пока нет. В планах сборка под Firefox (там MV3 отличается в деталях). [Голосуйте в issues.](https://github.com/stufently/free-zakupki-gov-ru-monitor/issues)

### Расширение работает, когда браузер закрыт?

Нет. Это ограничение Chrome MV3: service worker живёт только пока запущен браузер. Если нужен мониторинг 24/7 даже при выключенном браузере, используйте связку сервер плюс Telegram-бот. Такая интеграция тоже в планах, отдельной папкой в этом же репо.

### Как часто можно опрашивать RSS?

Минимум 1 минута (это ограничение `chrome.alarms`). По умолчанию стоит 10 минут. Можно поставить от 1 минуты до недели через настройки. Слишком частый поллинг теоретически попадает под rate limit zakupki.gov.ru, но на интервалах от 1 минуты у меня и тестировщиков претензий не было.

### RSS не открывается, фид пустой — что делать?

1. Проверьте URL в браузере: должен открыться XML с тегами `<channel>` и `<item>`.
2. Убедитесь, что в URL нет переносов строк — они часто ломаются при копировании из мессенджеров.
3. Проверьте, что zakupki.gov.ru в принципе доступен с вашей сети (бывают downtime).
4. Если фид валидный, но расширение не видит новых записей — это нормально для первого запуска. Расширение запоминает текущие записи и не считает их новыми. Уведомления начнут приходить со следующей проверки.

### Можно ли подключать любые RSS, не только zakupki.gov.ru?

В текущей версии `host_permissions` ограничены `zakupki.gov.ru` и его поддоменами. Расширение специально не претендует на доступ к произвольным сайтам, чтобы пройти review Chrome Web Store с минимальными требованиями. Парсер совместим с RSS 2.0 и Atom. В планах опциональный режим для сторонних RSS через `optional_host_permissions` и явное согласие пользователя. [Голосуйте в issues.](https://github.com/stufently/free-zakupki-gov-ru-monitor/issues)

### Где сохраняются данные?

Локально, в `chrome.storage.local`. Посмотреть содержимое: `chrome://extensions`, расширение, «Подробности», «Просмотр данных хранилища».

## Технические детали

- Manifest V3 (Chrome 88+ принимает только MV3 для новых расширений).
- Service worker (`background.js`) на нативных ES-модулях, без бандлера и сборки.
- Парсер RSS — нативный `DOMParser`, без зависимостей. Понимает RSS 2.0, Atom, namespaced теги (`dc:date` и т.п.).
- Хранилище — `chrome.storage.local`, квота около 10 МБ.
- Нет внешних HTTP-запросов, кроме fetch к указанным вами RSS-URL на zakupki.gov.ru.
- Минимальная версия Chrome — 119 (нужно для `DOMParser` в service worker).
- Нет фоновых сетевых запросов между интервалами проверки.
- Нет content scripts — расширение не вмешивается в открытые страницы.

Структура:

```
extension/
├── manifest.json      # MV3 manifest
├── background.js      # Service worker: alarms + fetch + parse + notify
├── rss.js             # Парсер RSS 2.0 / Atom
├── storage.js         # Обёртка над chrome.storage.local
├── options.html/.js/.css   # Страница настроек
├── popup.html/.js/.css     # Popup при клике на иконку
├── _locales/ru/messages.json
└── icons/             # 16/48/128 PNG иконки
```
